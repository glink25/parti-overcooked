# 新手上厨 — 全新地图机制版

2–4 人 Low-poly 3D 合作厨房游戏，可直接构建为 Parti 联机房间包。玩家需要在异形厨房、移动浮岛和动态机关间完成取菜、切菜、烹饪、装盘、上菜与洗碗。

![新手上厨游戏封面](public/cover-newbie-chefs.jpg)

## 核心玩法

- 派对模式连续挑战三张随机地图；无尽模式持续换图并管理食客怒气。
- 10 种食材、18 道菜谱，保留完整/切碎要求、炖煮、烧糊、脏盘和洗碗流程。
- 地图使用异形地块和独立分区；普通边界有实体墙保护，只有清楚标记的装卸口、裂谷与浮岛边缘会触发坠落。
- 原料、切好食材、空盘和成品菜均可掉落在地面或被投掷；传送带是与台面等高的封闭货运设备，货物只能经“入/出”接口台进出，玩家不能踩上带面。
- 地面与传送带物品存在 20 秒，最后 5 秒闪烁；盘子到期后会回到盘子循环，不会永久丢失。
- 玩家落入水域或空洞会丢失手持物，并回到自己在本轮地图上的出生点；移动平台出生点按复活时的平台位置计算。

## 投掷与接取

- 短按互动键执行拿取、放置、下锅、装盘和上菜。
- 手持物时按住互动键超过 0.3 秒进入蓄力，1.2 秒达到 5.5 格最大距离。
- 蓄力期间可以移动和改变朝向；松开后沿当前朝向投掷。
- 空手、正面朝向来物的队友会自动接住进入接取范围的物品。
- 投掷物可落到空台面、砧板或地面；带面不是投掷入口，落点命中带体时会以红色提示并退回附近安全地面。撞墙、撞关闭城门或落水会停止或消失。

## 六张地图

- **经典厨房**：有围墙的紧凑双环花园厨房，中央高台货运线通过专用输入、输出台提供备餐捷径。
- **一线天**：原料和切配集中在西浮岛，烹饪与出餐集中在东浮岛；中央渡台是唯一人员通路，交接台始终处于投掷距离内。
- **环岛餐吧**：外环负责原料和切配，中央岛负责烹饪、装盘与出餐；东西双输入台向循环带供货，中央南北双输出台接货，带体每 20 秒换向。
- **雪山餐车**：三座有护墙的工作区通过石道和短冰桥连接；两处冰裂洞具有深色断面、冰纹和实体绳索护栏，直线高台缆车负责跨区物流。
- **太空厨房**：封闭的左右备餐翼通过货运气闸向中央烹饪舱送货，玩家换舱必须走底部勤务通道。
- **城堡宴会厅**：中央四组三格城门每 16 秒随机换阵；外围勤务环道始终连通，门阵只改变更快的中央路线。

完整的逐图问题分析、协作目标和机关合同见 [`docs/map-design.md`](docs/map-design.md)。

## 操作

| 平台 | 移动 | 互动 / 投掷 | 切菜 / 洗碗 |
| --- | --- | --- | --- |
| 键盘 | WASD / 方向键 | 短按 E / 空格 / J 互动，长按蓄力投掷 | 按住 Q / F / K |
| 触屏 | 左半屏浮动摇杆 | 短按互动按钮，长按蓄力投掷 | 长按切/洗按钮 |

大屏使用更高、更宽且完全固定的整图镜头；只有空间明显不足的小屏才启用跟随。跟随镜头带有宽迟滞、移动死区和低速平滑，不随玩家朝向来回摆动。开局倒计时始终先展示地图全景。

## 全新架构

Worker 持有唯一权威状态并以 10Hz 更新：

```text
GameState
├── layout        terrain / platforms / stations / mechanisms
├── players       movement / carrying / charge / support / fall / roundSpawnSlot
├── worldItems    ground / airborne / conveyor
├── stations      counter / board / stove / conveyorPort state
├── platforms     authoritative offsets
└── mechanisms    conveyor / multi-cell gate / moving-platform state
```

地图机制通过统一注册表组合。移动平台使用本地坐标和稳定 ID；城堡门阵使用房间种子洗牌并提前预告下一通路；工位状态不依赖世界坐标字符串。客户端负责预测、插值、轨迹预览和渲染，所有实际互动、投掷命中、接取与危险区域判定均由 Worker 裁定。

## 本地开发

```bash
npm install
npm run dev
npm test
npm run test:maps
npm run verify
```

`npm run verify` 会先构建房间包，再运行全新的世界、机制、地图、客户端、做菜流程和产物测试。

构建产物位于 `dist/`：

```text
parti.room.json
index.html
room.worker.js
cover-newbie-chefs.jpg
```

浏览器打开 `dist/index.html#demo` 可查看由新 Worker 状态生成的演示画面。

## 发布

推送 `v*` tag 后按仓库现有 CI 构建并发布房间包。Worker 继续遵守 Parti 单文件约束，只导入 `defineRoom`；UI 继续构建为不依赖外部网络资源的单文件页面。
